What Am I Looking At?

- This is an artistic visualization of a tree graph generated by number enumeration algorithm utilizing digit mirroring.

- By using this site, you are creating Art with Math!

- Enjoy tuning the parameters to discover new patterns.

How is The Tree Generated?

- Each node represents a generated number.

- Blue nodes are prime numbers and red nodes are composite numbers

- A parent node is connected to it's children nodes with lines

- Child nodes are generated for each parent by rewritting the parent node's numerical value into every meaningful base, mirroring the resulting representaion and rewritting the new mirrored number back into base 10.

- For example: if the parent node is 11, 11 would be converted into base 2 resulting in 1011, this would be mirrored to 1101 in base 2 which is 13 in base 10. This would be repeated for bases 3 up to 10.

- Side note: An interesting result of this process is that prime numbers tend to generate relatively more prime numbers than composites. If you are a mathematician interested in researching this please connect with me via email or linkedin, I am looking for collaborators on this project.

What Do The Parameters Do?

- Seed Number: This will the root number used in the tree generation algorithm.

- Iteration: This is the number of times the algorithm will be applied. (Technically these graphs could extend forever)

- Max Child Nodes: This will limit the number of child nodes a parent node will have.

- Show Primes: This will show nodes that are prime.

- Show Composites: This will show nodes that are composite.
